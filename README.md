# 学習ローテーション

複数の科目を、毎日「前回の復習＋新しい学習」のワンセットで回すための、
1ファイルだけのブラウザアプリ。初期値は Illustrator / Photoshop / After Effects。

**公開URL: https://haizarakumin-collab.github.io/study-rotation/**
（ブラウザで開くだけで使えます。**APIキーの設定は不要** — 共有バックエンド経由で動きます。）

## 1日の流れ

1日 = 「前回やったものの復習」＋「新しいものの学習」。科目数の日数で一周する
（科目3つなら3日で一周）。

| 日 | 復習（前回） | 学習（新しい） |
|---|---|---|
| 1日目 | （前周の最後の科目） | 1つめの科目 |
| 2日目 | 1つめの科目 | 2つめの科目 |
| 3日目 | 2つめの科目 | 3つめの科目 |
| →繰り返し | | |

各日の「復習」は、前日に「学習」した科目と一致する。
科目は設定で自由に追加・削除・並べ替えでき、ローテーションは自動で組み直される。

## 使い方

1. `学習ローテーションを開く.command` をダブルクリックで起動
   （ローカルサーバー経由で開く。詳細は下記）。
2. 初回だけ「⚙️ 設定」で、科目ごとに名前・色・種類・URLを設定する。
   - **再生リスト**: 再生リストのURLを貼る。
   - **チャンネル（古い順）**: チャンネルのURLを貼る。全動画を古い順に並べる。
   - 「＋ 科目を追加」で科目を増やせる。↑↓で並べ替え、削除も可能。
   - 設定に **YouTube Data API の無料キー** を貼る（下記）。入力すると自動保存。
3. 各ブロックの「再生リスト／チャンネルを読み込む」を押すと動画一覧を取り込む。
4. 「学習」ブロックの「学習で見る動画：◯本目から」で開始位置を指定できる。
5. その日の復習・学習が終わったら「完了 → 次の日へ」。学習の本数が1つ進み、
   次に同じ科目が学習に回ってきたら自動で次の動画になる。

## 続けた記録・バックアップ

- 上部に「🔥 ○日連続／直近7日で○日／最後にやった日」を表示。「完了」を押した
  カレンダー日を記録する。
- 設定の「バックアップ」で、URL・キー・進捗・記録を1つのJSONに書き出し／読み込み。
  別のPCやブラウザに引っ越せる（localStorage はブラウザ単位の保存のため）。

## APIキーについて（基本は不要）

動画一覧の取得には YouTube Data API を使うが、キーは**共有バックエンド**
（[backend/](backend/) の Cloudflare Worker）がサーバー側に隠して持っているので、
**使う人はキー設定不要**。

- 自分のキーを使いたい上級者だけ、設定の「YouTube Data API キー（任意）」に入れると
  そのキーで直接取得する（共有の利用上限を気にせず使える）。
- バックエンドの仕組み・デプロイ手順は [backend/README.md](backend/README.md)。

## 起動（ローカルサーバー経由）

YouTubeの埋め込み動画は `file://`（ファイル直開き）では再生できず **エラー153**
になる。そのため `学習ローテーションを開く.command` は Python の簡易サーバーを
立ち上げ、`http://localhost:8765/index.html` で開く。

- ダブルクリックすると黒いターミナル窓が出る＝サーバー稼働中の印。使用中は閉じない。
- 使い終わったらその窓を閉じる（サーバーが止まる）。

## しくみ（メモ）

- 全部 `index.html` の中（HTML + CSS + JavaScript の1ファイル）。ビルド不要。
- 科目・進捗（何日目か／各科目を何本目まで学習したか）・URL・APIキー・記録は
  ブラウザの localStorage（`studyRotation.*`）に保存。同じURLで開けば続きから。
- ローテーションは科目配列から自動生成（day i = 学習: 科目i／復習: 科目i-1）。
- 再生リストは並び順そのまま、チャンネルは取得後に reverse して古い順にする。
  どちらも動画IDで1本ずつ `embed/動画ID` を埋め込む。
- 「新着を取り込む」で再取得しても、古い動画の位置は変わらないので進捗はズレない。
- スマホ幅（〜560px）向けのレイアウト調整あり。
